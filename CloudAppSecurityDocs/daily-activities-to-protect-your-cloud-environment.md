# <a name="daily-activities-to-protect-your-cloud-environment"></a>클라우드 환경을 보호하는 일상적인 활동
Cloud App Security를 실행하고 데이터 스트림 구성, 사용하도록 할 앱에 사용 권한 부여, 클라우드 환경을 모니터링할 정책 설정을 완료하면 Cloud App Security를 사용하여 클라우드를 제어 및 보호하고 위험을 관리할 수 있습니다.  
  
이 항목에서는 매일 수행해야 하는 작업을 설명합니다.  
  
## <a name="check-the-dashboard"></a>대시보드 확인  
Cloud App Security 포털을 열면 클라우드 환경의 미해결 경고, 활동 위반, 콘텐츠 위반, 사용자 활동이 발생한 지리적 위치를 그리는 지도를 제공하는 활동 지도, 연결된 앱 사용 추세에 대한 개요가 제공됩니다.  
  
매일 대시보드를 검사하여 트리거된 새 경고를 확인하고 처리하는 것이 좋습니다. 여기서 클라우드 환경의 상태를 감시하여 클라우드 환경 전체에서 발생하는 상황을 개괄적으로 파악할 수도 있습니다.  
  
![대시보드](./media/dashboard.png "dashboard")  
  
## <a name="handle-your-alerts"></a>경고 처리  
경고는 클라우드 환경을 보다 자세히 이해하기 위한 진입점입니다. 찾은 내용에 따라 새 정책을 만들 수도 있습니다. 예를 들어 관리자가 그린란드에서 로그온하는 것을 보고 나중에 그린란드에서 로그온하는 데 사용될 경우 관리자 계정을 자동으로 일시 중단하는 정책을 만들기로 결정할 수도 있습니다.  
  
모든 경고를 검토하고 정책을 수정하기 위한 도구로 사용하는 것이 좋습니다. 무해한 이벤트가 기존 정책에 대한 위반으로 간주되는 경우 불필요한 경고 수를 줄이기 위해 정책을 구체화해야 합니다.  
  
-   **미해결 경고**에서 **모든 경고 보기**를 클릭합니다.  
  
     이 경우 의심스러운 활동이나 설정된 정책 위반을 모두 볼 수 있으며, 클라우드 환경에 대해 정의한 보안 상태를 보호하는 데 도움이 됩니다.  
  
     ![경고](./media/alerts.png "alerts")  
  
-   각 경고에 대해 위반의 특성 및 필요한 응답을 조사하고 결정해야 합니다.  
  
     가장 중요한 항목이 먼저 처리되도록 **경고 유형** 또는 **심각도**를 기준으로 경고를 필터링할 수 있습니다.  
  
     특정 경고를 클릭합니다. 경고 유형에 따라 경고를 해결하기 전에 수행할 수 있는 다양한 작업을 수신됩니다.  
  
     경고를 조사할 때 처리해야 하는 다음 세 가지 유형의 위반이 있습니다.  
  
    -   **심각한 위반** - 즉시 응답해야 함  
  
         예제:  
  
         의심스러운 활동 경고의 경우 사용자가 암호를 변경할 때까지 계정을 일시 중단하는 것이 좋습니다.  
  
         데이터 누출의 경우 사용 권한을 제한하거나 파일을 격리하는 것이 좋습니다.  
  
         새로운 비사용 권한 서비스가 검색된 경우 프록시 또는 방화벽에서 해당 서비스에 대한 액세스를 차단하는 것이 좋습니다.  
  
    -   **의심스러운 위반** - 추가 조사가 필요함  
  
         사용자 또는 사용자의 관리자에게 활동의 특성을 문의할 수 있습니다.  
  
         추가 정보를 받을 때까지 활동을 미해결 상태로 둡니다.  
  
    -   **승인된 위반 또는 비정상적인 동작** - 합법적인 사용에서 발생함  
  
         경고를 해제합니다.  
  
-   이 프로세스를 완료하면 경고를 해결된 것으로 표시합니다.  
  
다음 표에서는 트리거될 수 있는 경고 유형 목록과 권장되는 해결 방법을 제공합니다.  
  
|경고 유형|설명|권장 해결 방법|  
|----------------|-----------------|----------------------------|  
|활동 정책 위반|이 유형의 경고는 만든 정책의 결과입니다.|- 이 유형의 경고를 대량으로 작업하려면 정책 센터 내에서 직접 작업하여 완화하는 것이 좋습니다.<br />- 더 많은 필터와 보다 세부적인 제어를 추가하여 불필요한 항목을 제외하도록 정책을 미세 조정합니다.<br />- 정책이 매우 정확하고 경고가 보증되었으며 즉시 중지하려는 위반인 경우 정책에 자동 수정을 추가하여 자동화된 응답으로 이동하는 것이 좋습니다.|  
|파일 정책 위반|이 유형의 경고는 만든 정책의 결과입니다.|- 이 유형의 경고를 대량으로 작업하려면 정책 센터 내에서 직접 작업하여 완화하는 것이 좋습니다.<br />- 더 많은 필터와 보다 세부적인 제어를 추가하여 불필요한 항목을 제외하도록 정책을 미세 조정합니다.<br />- 정책이 매우 정확하고 경고가 보증되었으며 즉시 중지하려는 위반인 경우 정책에 자동 수정을 추가하여 자동화된 응답으로 이동하는 것이 좋습니다.|  
|손상된 계정|이 유형의 경고는 Cloud App Security에서 손상된 계정(계정이 권한 없는 방식으로 사용되었을 가능성이 매우 높음)을 식별할 때 트리거됩니다.|사용자에 접근하고 사용자가 암호를 변경할 때까지 계정을 일시 중단하는 것이 좋습니다.|  
|비활성 계정|이 경고는 연결된 클라우드 앱 중 하나에서 계정이 더 이상 사용되지 않는 경우에 트리거됩니다.|사용자와 사용자의 관리자에게 문의하여 계정이 여전히 활성 상태인지 확인합니다. 그렇지 않으면 사용자를 일시 중단하고 앱의 라이선스를 종료합니다.|  
|새 관리자|연결된 앱에 대한 권한 있는 계정의 변경 내용을 경고합니다.|새 관리자 권한이 실제로 사용자에게 필요한지 확인하고, 필요하지 않을 경우 노출을 줄이기 위해 관리자 권한을 취소하는 것이 중요합니다.|  
|새 관리자 위치|연결된 앱에 대한 권한 있는 계정의 변경 내용을 경고합니다.|이 비정상적인 위치에서의 로그온이 합법적인지 확인하고, 합법적이 아닌 경우 노출을 줄이기 위해 관리자 권한을 취소하거나 계정을 일시 중단하는 것이 좋습니다.|  
|새 위치|새 위치에서 연결된 앱에 대한 액세스와 관련된 정보 제공용 경고이며, 국가별로 한 번만 트리거됩니다.|특정 사용자의 활동을 조사합니다.|  
|새 검색된 서비스|Cloud Discovery에서 새 앱이 검색되었다는 섀도 IT에 대한 경고입니다.|<ul><li>앱 카탈로그를 기반으로 하여 서비스의 위험을 평가합니다.</li><li>활동을 드릴다운하여 사용 패턴과 전파를 파악합니다.</li><li>앱에 사용 권한을 부여할지 여부를 결정합니다.<br /><br />     비사용 권한 앱:<br /><br /> <ul><li>프록시 또는 방화벽에서 사용을 차단하는 것이 좋습니다.</li><li>승인되지 않았으며 동일한 범주에 사용 권한 앱이 있는 경우 비사용 권한 앱의 사용자 목록을 드릴다운하고 내보낸 다음 사용 권한 앱으로 마이그레이션하도록 요청할 수 있습니다.</li></ul></li></ul>|  
|의심스러운 활동|이 경고를 통해 조직의 예상 활동이나 사용자와 일치하지 않는 비정상적인 활동이 검색된 것을 알 수 있습니다.|동작을 조사하고 사용자에게 확인합니다.<br /><br /> 이 유형의 경고는 사용자 환경을 자세히 확인하고 이러한 경고를 통해 새 정책을 만들기 시작하는 데 적합합니다. 예를 들어 누군가가 연결된 앱 중 하나에 많은 양의 데이터를 갑자기 업로드하는 경우 해당 유형의 비정상적인 동작을 제어하는 규칙을 설정할 수 있습니다.|  
|의심스러운 클라우드 사용|이 경고를 통해 조직의 예상 활동이나 사용자와 일치하지 않는 비정상적인 활동이 검색된 것을 알 수 있습니다.|동작을 조사하고 사용자에게 확인합니다.<br /><br /> 이 유형의 경고는 사용자 환경을 자세히 확인하고 이러한 경고를 통해 새 정책을 만들기 시작하는 데 적합합니다. 예를 들어 누군가가 연결된 앱 중 하나에 많은 양의 데이터를 갑자기 업로드하는 경우 해당 유형의 비정상적인 동작을 제어하는 규칙을 설정할 수 있습니다.|  
|개인 계정 사용|이 경고를 사용하면 새 개인 계정이 연결된 앱의 리소스에 액세스할 수 있음을 알 수 있습니다.|외부 계정에서 사용자의 공동 작업을 제거합니다.|  
  
## <a name="use-policies-to-assess-risk"></a>정책을 사용하여 위험 평가  
미해결 경고를 살펴본 후 **정책 센터**로 이동하여 경고를 트리거하지 않은 정책 위반을 검토합니다.  
  
-   Cloud App Security 포털에서 **제어**, **정책**을 차례로 클릭합니다.  
  
-   특정 정책을 클릭하여 경고를 트리거하지 않은 정책 일치 항목의 **지금 위반 중** 목록을 확인합니다.  
  
-   한 번에 하나씩 위반을 클릭하고 각각에 대해 수행할 작업을 결정합니다. 거버넌스 작업에 대한 자세한 내용은 다음을 참조하세요.  
  
     예를 들어 정책이 규정 준수 위반을 찾도록 설정되었으며 누군가가 신용 카드 번호를 OneDrive의 파일에 저장하는 경우 정책에 일치 항목이 포함됩니다.  
  
     ![pci 일치](./media/pci-matches.png "pci matches")  
  
-   일치 항목을 클릭하여 정책을 위반한 실제 파일을 확인합니다.  
  
     ![pci 콘텐츠 일치](./media/pci-content-matches.png "pci content matches")  
  
     파일 자체를 클릭하면 파일에 대한 정보를 가져올 수 있습니다.  
  
     **공동 작업자**를 클릭하면 이 파일에 대한 액세스 권한이 있는 사용자를 볼 수 있습니다.  
  
     **일치 항목**을 클릭하면 실제 신용 카드 번호를 볼 수 있습니다.  
  
     ![콘텐츠 ccn 일치](./media/content-matches-ccn.png "content matches ccn")
